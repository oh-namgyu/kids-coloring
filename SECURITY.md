# Security Policy

## Reporting a vulnerability

Please report security issues privately using GitHub's
[private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability):
open the repository's **Security** tab → **Report a vulnerability**.

Please do not open a public issue for security reports.

## Scope

This is a fully static, client-side web app with **no backend, no accounts, and no data collection** —
all drawing happens locally in the browser and nothing is sent to a server. The main areas of interest
are the build/dependency supply chain and the service worker caching behavior.

We aim to acknowledge valid reports within a reasonable time and address them as quickly as is practical.
